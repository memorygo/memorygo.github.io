Node.js

##引言

为什么JavaScript 可以在浏览器中被执行

不同浏览器使用不同的JavaScript解析引擎

  • Chrome 浏览器 => V8
  • Firefox 浏览器 => OfinMonkey(奥丁猴)
  • Safri 浏览器 => JSCore
  • IE 浏览器 => Chakra (查克拉1)

每个浏览器都内置了DOM、BOM这样的API函数，因此，浏览器中的JavaScript可以通过对应的api调用它们。

浏览器中JavaScript的运行环境

JavaScript运行环境大致上由

  • JS解析引擎

  • 内置的api接口

构成的。

可以说

浏览器就是一个JavaScript的运行环境，JavaScript用浏览器作为运行环境，属于前端开发

而JavaScript可以通过node作为运行环境就可以做后端开发

什么是Node.js

是一个基于 Chrome V8 引擎的 JavaScript 运行环境

Node.js 可以做什么

Node.js本身只提供基础的功能和API。但是基于Node.js的基本功能，涌现出来很多功能强大的工具和框架。

  • Express 框架，快速构建Web应用

  • Electron框架,可以构建跨平台的桌面应用

  • restify框架，可以快速构建API接口项目

    ​

npm与包

Node.js中的第三方模块，又叫做包

内置模块 用户自定义模块 第三方模块

包是基于内置模块封装出来的，提供了更高级，更方便的API,提高了开发效率。

包的下载

npm.inc公司旗下有着全球最大的包共享平台https://www.npmjs.com/我们可以在里面检索我们需要的包。

npm.inc公司还有专门下载包的服务器https://registry.npmjs.org/，我们可以通过这个网址拉取自己需要的包。

npm（Node Package Manager）则是该公司提供的一个包管理工具，用于把https://registry.npmjs.org/上的包下载到本地使用

安装Node.js时，npm已经被一起安装了。

npm 初次下载包后

目录文件夹会多出node_modules文件夹和package-lock.json配置文件

  • node_moules, 用来存放安装到项目中的包
  • package-lock.json，记录 node_moudles 目录下每个包的下载信息，如包的名字，版本号，下载地址

项目根目录中，必须提供包管理配置文件package.json，来记录项目配置信息

  • 项目名称，描述，版本号等
  • 用到哪些包
  • 哪些包只在开发用到，哪些包在开发和部署都需要

多人协作时

第三方包的体积占项目的绝大部分内存，所以共享代码时需剔除node_modules，

然后根据package.json和package-lock.json,本地安装自己需要的包

创建项目时

package.json是必须要有的，npm提供了快捷命令，为我们自动创建了package.json配置文件的空白模版

此后，我们npm install 包 后，下载来的包的信息，会自动录入 package.json 文件中

拉取项目时

我们可以根据拉取来的项目的package.json文件 一次性下载所有的包。

devDependencies 节点

在package.json配置文件中，有着dependencies节点和devdependencies节点

有些包只在开发时会用到，项目上线后不会再用到，建议将此类包记录到 devDepencies节点中。

有的包在开发和上线都需要用到，建议记录到dependencies节点。
