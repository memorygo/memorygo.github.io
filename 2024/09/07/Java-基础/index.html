jdk的组成

  • JVM (Java Virtual Machine): Java 虚拟机，真正运行Java程序的地方
  • 核心类库：Java自己写好的程序，给程序员调用

JVM 和 核心类库 的 集合 叫做 JRE （Java Runtime Environment): Java 的运行环境

  • 开发工具： java.exe , javac.exe …

运行java程序过程

  • 通过javac.exe 运行 Hello.java
  • 产生Hello.class
  • 通过java.exe 将Hello.class放入JVM中，在JVM中调用核心类库

Java 跨平台原理

  • 一次编译处处可用。

    各个平台 只需安装 JVM 虚拟机 就可运行

编码

ASCII

全称 American Standard Code for Information Interchange,美国信息交换标准码表
只有英文和字符，’a’对应97

GB2312 编码

1981.5.1 发布的简体中文汉字编码国家标准。收录7445个图形字符，其中包括6763个汉字。

BIG5 编码

1984年发布，台湾地区繁体中文标准字符集，共收录13053个中文字。

GBK编码

2000年发布，收录21003个汉字，包含国家标准GB13000-1中的全部中日韩 和 BIG5编码中的所有汉字。

Unicode编码

国际标准字符集，又称 ”万国码“，世界各种语言的每个字符定义一个唯一的编码，以满足跨语言，跨平台的文本信息转换。

原码、反码、补码

原码

十进制数据的二进制表现形式，最左边是符号位，0为正，1为负。

原码 在正数计算中没有问题，负数计算时要出错

反码

正数的反码不变，负数的反码时原码的符号位不变，数值取反。

反码是为了解决原码不能计算负数的问题而出现的。

但是当负数运算时跨0，会与实际值有1的误差。

补码

正数的补码不变，负数的补码在反码的基础上 +1。

补码是为了解决负数计算时跨0的问题

补码还多记录了一个特殊值-128，数据在1个字节中，没有原码、反码。

注意：计算机中的存储和计算都是以补码的形式进行的。
